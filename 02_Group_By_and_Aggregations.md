# 📊 Pattern 2: GROUP BY & Aggregations (Questions 11–18)

Aggregation functions summarize data into meaningful insights. They are among the most frequently asked SQL interview topics.

## Topics Covered

- **GROUP BY** – Groups rows with the same values into a single group for performing aggregate calculations.
- **HAVING** – Filters grouped data after the `GROUP BY` operation based on aggregate conditions.
- **COUNT()** – Returns the total number of rows or non-NULL values in a group.
- **SUM()** – Returns the total sum of numeric values in a column or group.
- **AVG()** – Returns the average (mean) value of a numeric column or group.
- **MIN()** – Returns the smallest value from a column or group.
- **MAX()** – Returns the largest value from a column or group.
- **Conditional Aggregation** – Uses aggregate functions with `CASE` expressions to calculate values based on specific conditions (e.g., `SUM(CASE WHEN status = 'Approved' THEN amount ELSE 0 END)`).

# Q11. Find Total Revenue Generated by Each Product

### Difficulty

🟢 Beginner

### Concept

Calculate the total revenue earned by each product.

### SQL Server Solution

```sql
SELECT
    p.product_name,
    SUM(o.amount) AS TotalRevenue
FROM orders o
INNER JOIN products p
    ON o.product_id = p.product_id
GROUP BY
    p.product_name
ORDER BY
    TotalRevenue DESC;
```

### Explanation

- GROUP BY creates one row for each product.
- SUM() calculates total revenue.

### Expected Output

|Product|Revenue|
|--------|--------|
|Laptop|7500|
|Keyboard|2800|

### Interview Tip

Every non-aggregated column must appear in the GROUP BY clause.

# Q12. Count Total Orders Placed by Each Customer

### Difficulty

🟢 Beginner

### SQL Server Solution

```sql
SELECT
    c.customer_id,
    c.name,
    COUNT(o.order_id) AS TotalOrders
FROM customers c
LEFT JOIN orders o
    ON c.customer_id = o.customer_id
GROUP BY
    c.customer_id,
    c.name
ORDER BY
    TotalOrders DESC;
```

### Explanation

LEFT JOIN ensures customers without orders are also included.

### Expected Output

|Customer|Orders|
|---------|------|
|Alice|8|
|Bob|5|

### Performance Tip

COUNT(column_name) ignores NULL values.

# Q16. Find Departments Having More Than 5 Employees

### Difficulty

🟡 Intermediate

### SQL Server Solution

```sql
SELECT
    department,
    COUNT(*) AS EmployeeCount
FROM employees
GROUP BY department
HAVING COUNT(*) > 5;
```

### Explanation

HAVING filters groups after aggregation.

### Interview Tip

WHERE filters rows.

HAVING filters groups.

# Q17. Find Customers Whose Total Purchase Exceeds 1000

### Difficulty

🟡 Intermediate

### SQL Server Solution

```sql
SELECT
    c.customer_id,
    c.name,
    SUM(o.amount) AS TotalPurchase
FROM customers c
INNER JOIN orders o
    ON c.customer_id = o.customer_id
GROUP BY
    c.customer_id,
    c.name
HAVING SUM(o.amount) > 1000
ORDER BY TotalPurchase DESC;
```

### Explanation

HAVING is used because SUM() is an aggregate function.

### Expected Output

|Customer|Purchase|
|---------|---------|
|Alice|5600|
|Bob|2100|

# Q18. Monthly Sales Summary

### Difficulty

🟡 Intermediate

### SQL Server Solution

```sql
SELECT
    YEAR(order_date) AS SalesYear,
    MONTH(order_date) AS SalesMonth,
    COUNT(*) AS TotalOrders,
    SUM(amount) AS TotalSales,
    AVG(amount) AS AverageOrderValue
FROM orders
GROUP BY
    YEAR(order_date),
    MONTH(order_date)
ORDER BY
    SalesYear,
    SalesMonth;
```

### Explanation

YEAR() and MONTH() extract date parts in SQL Server.

### Expected Output

|Year|Month|Orders|Sales|
|----|-----|------|------|
|2025|1|18|24500|
|2025|2|22|28100|

# 🎯 GROUP BY Cheat Sheet

| Function | Purpose |
|----------|---------|
| COUNT() | Count rows |
| SUM() | Total value |
| AVG() | Average value |
| MIN() | Minimum value |
| MAX() | Maximum value |
| GROUP BY | Group records |
| HAVING | Filter grouped records |
